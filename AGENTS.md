# AGENTS.md

## What this repo is

A personal macOS dotfiles tree mounted at `~/.config`. The `README.md` lists the tools at a high level (tmux, ghostty, zsh+ohmyposh, nvim, aerospace, sketchybar, jankyborders, raycast). There is no build / test / lint pipeline Рђћ work happens by editing configs and reloading the live tool.

## Commits

**The user commits all changes manually. Never run `git commit`/`git add`/push in this repo Рђћ not even when work is finished or verified.** Make the edits, report what changed, and stop; the user stages and commits themselves.

## Source-of-truth files (not obvious from directory layout)

- **`COLORS.md`** Рђћ palette guide and the canonical source of truth for every color used across tools. Lists semantic roles (`canvas`, `surface-active`, `surface-chrome`, `surface-highlight`, `divider-subtle`, `text`, `text-muted`, accent-*) with hex values and per-tool usage. Also has a `Cross-tool consistency` section flagging known near-misses, a `Claude Code integration` section documenting the custom theme, and a `Git colors (cross-tool)` table. **When changing any color anywhere, consult this file first.** When introducing a new color, add or extend a role rather than scattering inline hex.
- **`ghostty/themes/vague`** Рђћ the base color palette file. **Do not edit this when adjusting the user's effective palette** Рђћ instead override the specific keys (`background`, `foreground`, etc.) in `ghostty/config`. Keeps the theme file reusable.
- **`tmux_scripts/`** Рђћ referenced from `.tmux.conf` for popup / session / pane management. Not a tool directory; it's a script bundle.
- **`tmux_scripts/tsave` + `tmux_scripts/tload`** Рђћ session snapshot/restore. `tsave [name]` walks every session Рєњ window Рєњ pane to `~/.config/tmux_sessions/<name>.{json,md}` (the `.md` is the human-readable `SESSIONS:` tree; Claude Code panes are recorded with their `sessionId`, recovered via the pane's process tree Рєњ `claude/sessions/<pid>.json`). `tload <name|path>` recreates them Рђћ skipping sessions that already exist, restoring pane geometry from the saved `window_layout` string, and relaunching only `claude` panes via `claude --resume <id>` (every other pane returns as a plain shell in its saved cwd, no command replay). Snapshots live in `~/.config/tmux_sessions/`, gitignored via `.git/info/exclude` (not the tracked `.gitignore`).
- **`aerospace/aerospace.toml`** Рђћ tiling WM config; also the launcher of `sketchybar` + `borders` (via `after-startup-command`). Keybinds route through shell scripts in `aerospace/` (`keyswap.sh`, `cycle_workspace_window.sh`, `move_workspace_cycle.sh`, `toggle_gaps.sh`, `toggle_keyswap.sh`) and Hammerspoon (`hs -c "..."`). Workspaces `1*`РђЊ`5*` are force-assigned to `main`; `1^`/`2^` to secondary (`dell`/`built-in`).
- **`sketchybar/sketchybarrc`** Рђћ the bar config; items + event subscriptions are defined inline. Plugins in `sketchybar/plugins/` are executed by sketchybar via `script=`. The bar is launched by AeroSpace's `after-startup-command`, not by a launchd/brew service.
- **`hammerspoon/window.lua`** Рђћ all window sizing/float logic; frames computed directly (no Rectangle). `init.lua` exports the functions as `_G` globals so AeroSpace callbacks and `hs -c "..."` can call them by name.
- **`hammerspoon/bar.lua`** Рђћ SketchyBar profile toggle (`toggleMode`), reload (`reload`), and AeroSpace server toggle (`toggleAerospace`); also shrinks `hs.alert.defaultStyle` globally. Exported as `_G.toggleBarMode` / `_G.toggleAerospace`.
- **`sketchybar/sketchybarrc.full` + `sketchybar/sketchybarrc.performance`** Рђћ the two bar profile sources of truth (full vs. performance). The bare `sketchybarrc` is a copy that gets overwritten on each `cmd-ctrl-alt-p` toggle Рђћ always edit the `.full`/`.performance` files directly. `toggle_mode.sh` does the copy + flag flip.
- **`KEYBINDS.md`** Рђћ full reference of every active keybind across AeroSpace, Hammerspoon, and tmux (core binds + plugin/integration sub-keys). **When you add/remove/rebind a key, update this file to match.** Organized by tool Рєњ function group, with root vs. prefix scope noted for tmux binds.

## Cross-tool palette discipline (the main "architecture")

The palette is enforced *by convention*, not tooling. Eight surfaces share colors:

- Ghostty (`ghostty/config` overrides + `themes/vague` ANSI palette)
- tmux (`.tmux.conf` Рђћ `@color-*` user options + inline hex)
- zsh prompt (`ohmyposh/base.json` Рђћ segment-by-segment hex)
- nvim theme (`nvim/lua/plugins/vague.lua` Рђћ `colors` override + `on_highlights`)
- nvim chrome (`nvim/lua/plugins/lualine.lua`, `nvim/lua/plugins/dropbar.lua`, `nvim/lua/plugins/devicons.lua`)
- nvim diagnostics / gitsigns (via `vague.lua` `on_highlights`)
- nvim FoldColumn / NonText / FloatBorder / render-md heading backgrounds (also via `vague.lua` `on_highlights`, see the `surface-fold`, `surface-extend`, `surface-heading-h1/h2/h3`, `text-ui` roles in `COLORS.md`)

When a user asks to change a color, the change usually touches **2РђЊ4 of these surfaces**. Default workflow:

1. Identify the affected role(s) in `COLORS.md`.
2. Find every usage via `grep -rn '<hex>' ~/.config` (excluding `.git`, `claude/plans`, `claude/projects`, `nvim/lazy-lock.json`).
3. Edit all sites in one pass and update `COLORS.md` to reflect the new value or new role.
4. Hex case is *not* normalized Рђћ files have a mix of `#1c1c24` and `#1C1C24`. Don't normalize unprompted; both are functionally identical.

## Bar + window-management stack (AeroSpace / SketchyBar / Hammerspoon)

Three tools are wired together: AeroSpace emits callbacks on workspace/focus change Рєњ SketchyBar renders the bar Рєњ Hammerspoon does all window sizing/float (AeroSpace only tiles; Hammerspoon owns floating-window geometry).

**AeroSpace callback syntax differs per key (`aerospace/aerospace.toml`).** This is a recurring footgun:
- `exec-on-workspace-change` (and other `exec-on-*-change` callbacks) take an **external process argv** array: `['/bin/bash', '-c', 'Рђд']`. Do **not** prefix with the `exec-and-forget` AeroSpace command token Рђћ that is not a binary, and adding it silently breaks the callback. The callback runs sync; keep it cheap.
- `on-focus-changed` and keybinding entries take the **AeroSpace command string** form: `'exec-and-forget sketchybar --trigger Рђд'` (single TOML string, `exec-and-forget` is a real command here). Binding entries can also be arrays mixing AeroSpace commands and `exec-and-forget` strings.
When in doubt, check the AeroSpace guide: callbacks spawn a process from argv; bindings run AeroSpace commands.

**SketchyBar events drive the bar (`sketchybar/sketchybarrc` defines 3 custom events):**
- `aerospace_workspace_change` Рђћ fired by AeroSpace's `exec-on-workspace-change` with `FOCUSED_WORKSPACE`; subscribers: `ws_1` (Рєњ `frontapps.sh`) and `space.1^` (Рєњ `space_colors.sh`).
- `aerospace_focus_change` Рђћ fired by `on-focus-changed`; subscriber: `ws_1` (Рєњ `frontapps.sh`).
- `aerospace_keyswap_change` Рђћ fired by `toggle_keyswap.sh`; subscriber: `keyswap` item.
- **Never use `sketchybar --update`.** It forces *every* script to run at once; on monitor-attach (SketchyBar destroys/recreates all bars) this multiplies into heavy fanout and visible lag. Initial render uses targeted `--trigger` calls instead (end of `sketchybarrc`). If you add an item whose script must run at startup, add a corresponding `--trigger`, not `--update`.
- **`frontapps.sh` is the heavy plugin** (~2 aerospace IPC queries + awk/sort + a borders-color diff per run). It has a leading-edge coalescing lock (`mkdir` atomic lock + pid guard + dirty flag) because a workspace switch fires both `aerospace_workspace_change` and `aerospace_focus_change` concurrently Рђћ don't bypass the lock or you'll re-introduce the double-run race. It rebuilds up to 7 workspace slots ├Ќ 6 app slots + separators in one batched `sketchybar` call.
- **`space_colors.sh` is the single driver for all `space.*` indicator colors** Рђћ only `space.1^` subscribes to the event; the script batch-colors every `space.*` item in one call. When you add/remove a center `space.*` item, update the sid list in `space_colors.sh` to match or that item's color goes stale.
- **`sketchybar/plugins/aerospace.sh` is legacy/dead** Рђћ it was the old per-item color script replaced by the batched `space_colors.sh`. It is not wired to any active item; don't wire it back.
- **Center block layout is `* *-*-* *`**: workspace indicators are still `1^ 1* 2* 3* 2^`, but the middle three are visually connected (`*-*-*`) and there is **no overline**. The connectors are labels (`-`) on `space.1*`/`space.2*` with a fixed inactive color; active state is icon-only (`space_colors.sh` recolors `icon.color` only), so only the star highlights. `4*`/`5*` are intentionally NOT rendered as center indicators but remain reachable via keyboard (`cmd-4`/`cmd-5`) and shown in the left block when occupied. Don't re-add `4*`/`5*` to the center loop or to `space_colors.sh` without being asked. `space.1^` still subscribes to `aerospace_workspace_change` and drives `space_colors.sh` for all five.
- **Don't batch the right-block items into one `sketchybar` call.** The left block (one big `CMD` array of ~84 items) batches fine, but batching the right block (`battery` / `sep_right` / `date`) into a single `--add ... --set ... --add ... --set ...` call causes the right-block items to stack on top of the center block. Keep the right block as 3 separate `sketchybar --add item ... --set ...` calls (the form that works). The centerРєњright gap is set by `icon.padding_right=16` on `space.2^` (the rightmost center star); adjust that single value to tune the gap.
- **Performance mode Рђћ two decoupled keybinds, driven by `hammerspoon/bar.lua`:**
  - `cmd-ctrl-alt-p` (`bar.toggleMode`) Рђћ swaps the active profile by *copying* the chosen profile over `sketchybarrc` + flips `/tmp/sketchybar_perf_mode`. **Does NOT reload** Рђћ intentionally decoupled so you can stage a swap and apply it on your own timing.
  - `cmd-ctrl-alt-o` (`bar.reload`) Рђћ runs `sketchybar --reload` (the sole reload path).
  - Profiles: `sketchybarrc.full` (full setup Рђћ source of truth, edit this) and `sketchybarrc.performance` (right-block-only: date + battery, `blur_radius=0`, no AeroSpace events, renders on ALL displays Рђћ no `display=` key so it uses sketchybar's `all` default). Copy, not symlink Рђћ sketchybar reloads a plain file unambiguously. `toggle_mode.sh` does the copy + flag flip and echoes the new mode; Hammerspoon shows a lowercase `hs.alert` (`bar: full` / `bar: performance`).
  - The `/tmp/sketchybar_perf_mode` flag also gates the AeroSpace callbacks in `aerospace.toml` (`exec-on-workspace-change` / `on-focus-changed` no-op while the flag exists) so workspace/focus switches incur zero bar IPC in perf mode.
  - On the first toggle, if `sketchybarrc.full` is missing the script migrates it from the current `sketchybarrc`. **Always edit `sketchybarrc.full` / `sketchybarrc.performance` directly** Рђћ the bare `sketchybarrc` is just a copy that gets overwritten on each toggle.
  - `hs.alert.defaultStyle` is shrunk globally in `bar.lua` (textSize=14, radius=8) so alerts render as small pills, not the default huge centered overlays.

**Hammerspoon window sizing (`hammerspoon/window.lua`):**
- `init.lua` exports `_G` globals so `hs -c "<fn>()"` works from AeroSpace: `almostMaximize`, `almostMaximizeAfterMove`, `center`, `center43`, `almostMaximizeAll`, `almostMaximizeFocusedWorkspace`, `fullMaximize`, `toggleFloatKeepPos`, plus `toggleBarMode` and `toggleAerospace` from `bar.lua`. When you add a window function that AeroSpace calls, export it here.
- **`cmd-ctrl-alt-*` keybinds** (all bound in `init.lua`): `c` = `almostMaximizeFocusedWorkspace` (workspace-wide, display-wide fallback), `x` = `almostMaximize` (big-gap inset), `b` = `center43` (4:3 centered, smaller than `x`; `scale=0.8` knob in `window.lua`), `v` = `maximize` (small-gap inset), `shift-v` (= `cmd-ctrl-alt-shift-v`) = `fullMaximize` (same left/right as `v`, top inset = bottom inset so the window covers the SketchyBar strip Рђћ distinct from `v` which uses a larger bar-clearing top), `space` = `center` (no resize), `p` = bar profile swap (no reload), `o` = bar reload, `shift-o` (= `cmd-ctrl-alt-shift-o`) = `toggleAerospace` (enables/disables the AeroSpace server). Don't rebind these without checking AeroSpace `hs -c "..."` callers don't depend on the same combo.
- **PATH gotcha:** Hammerspoon is a GUI app; `os.execute`/`hs.execute` run with an env that typically lacks `/opt/homebrew/bin` in `PATH`. A bare `sketchybar` or `aerospace` in those calls is silently "command not found" (often hidden behind `2>/dev/null`). Always use absolute paths (`/opt/homebrew/bin/sketchybar`, `/opt/homebrew/bin/aerospace`) Рђћ this is the convention in `window.lua` and `bar.lua`. Same applies to shell scripts Hammerspoon spawns via `/bin/bash <script>`.
- **`almostMaximizeFocusedWorkspace` (cmd-ctrl-alt-c) has a display-wide fallback.** When AeroSpace is active it floats + snaps every window on the focused workspace (the original behavior). When AeroSpace is disabled/paused/uninstalled it instead snaps every standard visible window on the focused window's display (no AeroSpace IPC, no float step Рђћ windows are already free-floating without the WM); if there's no focused window it snaps all visible standard windows. Detection is via an `aerospace list-windows --all` probe (`aerospaceAvailable`), not by empty output (an empty workspace vs. a disabled server are indistinguishable by output alone).
- Two inset presets mirror AeroSpace gaps: `ALMOST` (big gaps) and `MAXIM` (small gaps). **These must stay in sync with `toggle_gaps.sh`'s `BIG_*`/`SMALL_*` constants** Рђћ both encode the same two gap states for floating vs tiled windows. `toggle_gaps.sh` rewrites the `[gaps]` table in `aerospace.toml` via `sed` then `aerospace reload-config`; the Hammerspoon presets are hardcoded px values matching the same geometry.
- **`almostMaximizeAfterMove()` is a debounced snap**, used only for cross-monitor moves (`cmd-shift-w`). The Accessibility API can report the *old* screen for a few ms after `move-node-to-monitor --focus-follows-window`, so an immediate `snapInsets()` computes the frame against the wrong monitor and the window lands off-center/on the wrong display. It polls the focused window's `screen():id()` until stable (or ~250ms timeout). Don't replace it with the immediate `almostMaximize()` for cross-monitor binds. Within-monitor binds (`cmd-ctrl-1..5`, `move_workspace_cycle.sh`) don't need it Рђћ `1*..5*` and `1^..2^` are each force-assigned to a single monitor, so screen identity can't change.
- **`rectangle-keybinds.lua` is disabled** (the `require` is commented in `init.lua`); window sizing is pure Hammerspoon via `window.lua`. Don't re-enable it.

## nvim plugin internals (non-obvious)

- **Trouble keymaps route through `nvim/lua/utils/trouble.lua`, not `require("trouble").toggle` directly.** All the `<leader>x*` mappings in `whichkey.lua` call `trouble_helpers.toggle(mode)`, which calls `M.open(mode)` Рєњ `view:wait()` + `view:fold_level({ level = N })` for mode-specific initial fold depth (symbols/lsp = 3, diagnostics/qflist = 2). If you add a new trouble keymap and bypass the helper, you lose the auto-fold behavior. If you add a new mode, add its fold level to the `fold_levels` table in the helper.
  - `M.open` defaults `focus = true` (keybind opens focus the window). `<leader>xt` position toggle in `trouble.lua` explicitly passes `{ focus = false }` to keep cursor in the editor. Don't change the helper default without checking `<leader>xt` still passes `focus = false`.
  - `position_pairs` in `trouble.lua` is an **ordered array** iterated with `ipairs`, not `pairs()` over a table Рђћ Lua table iteration order is non-deterministic and was causing inconsistent `<leader>xt` behavior when multiple Trouble modes were open. Keep it as an ordered array.
  - trouble v3 does **NOT** emit a `User TroubleOpen` event. An earlier config used a `User TroubleOpen` autocmd to fold-on-open; it was dead code and has been removed. Don't reintroduce it.
- **Noice cmdline UI hides the native `:` cursor.** `noice.lua` currently has `cmdline.enabled = true, view = "cmdline"`. Noice renders the cmdline itself (`noice/ui/cmdline.lua`), which sometimes leaves no visible cursor when pressing `:`. If the user reports the cursor is gone, set `cmdline.enabled = false` in `noice.lua` to restore the native cmdline (messages and popupmenu still use Noice). Don't disable `messages.enabled` Рђћ that's a separate concern.
- **`vague.lua` `NonText` override has cascading side effects.** `listchars.extends`/`precedes` use the `NonText` highlight group, which is also inherited by path-like UI surfaces, end-of-buffer tildes, and other nontext elements. Setting `hl["NonText"] = { fg = ..., bold = true }` to make `┬╗/┬Ф` more visible will recolor all of those. The decision was to **not** override `NonText` and instead rely on the glyph choice in `listchars` alone. The override line is left commented in `vague.lua` (line ~101) with a one-line note; the dim reset groups (`BlinkCmpGhostText`, `LspInlayHint`, `GitSignsCurrentLineBlame`, `ComplHint`) below it are intentional restoration styling and should stay even if the override stays commented.
- **render-markdown keeps fence/dash lines visible by config, not by default.** `render-md.lua` sets `code.conceal_delimiters = false`, `code.border = "none"`, `code.background_inset = 0`, and `dash.enabled = false`. The plugin defaults conceal ` ``` ` fence lines, replace `---` with a virtual text line, and skip the fence lines from the background highlight Рђћ all of which the user has explicitly overridden. If you're asked to "make code blocks look nicer" or "tweak rendering," don't reset these to defaults; the user has already decided raw ` ``` ` and `---` lines should stay visible. `background_inset = 0` extends the `RenderMarkdownCode` background to cover the fence lines so they match the code body.
- **`<leader>pl` preview float is a non-focusable, close-on-action float.** Defined in `vim-keymaps.lua`, tracked via a module-local `preview_float_win` variable. It is opened with `enter = false`, `focusable = false`, anchored to the cursor (`relative = "cursor"`), and closes on any next action (`CursorMoved` / `InsertEnter` / `BufLeave` autocmd, `once = true`). Buffer-local `q`/`<Esc>` maps don't work because the float never takes focus Рђћ the close-on-action autocmds are the only dismissal path. The line content is trimmed of leading indentation via `:gsub("^%s+", "")` before display width calculation and buffer write. Re-pressing `<leader>pl` closes any existing preview first.

## Notable conventions you'll meet

- **tmux dynamic styling via `command-alias` + hooks.** `.tmux.conf` defines `refresh-active-bg` (a `set -g command-alias[200] 'Рђд'`) and wires it to 5 hooks (`after-resize-pane`, `window-layout-changed`, `after-new-window`, `after-select-window`, `pane-exited`). This is the workaround for tmux *style options* not supporting `#{?Рђд}` format expansion. If you need similar dynamic behavior on another style option, follow the same pattern: alias + hooks, not inline format expressions.
- **tmux style vs format options.** `pane-border-format` accepts `#{}` and inline `#[fg=Рђд]`; `window-active-style`, `pane-active-border-style`, and similar style options do not re-evaluate `#{?Рђд}` per draw. Don't try to embed conditionals in style options.
- **tmux reload only SETS, never UNSETS.** `prefix r` (`source-file`) re-runs `.tmux.conf` but does **not** remove hooks/options you've since deleted from the file, and per-pane `set -p` overrides persist regardless. After removing a `set-hook`/option, clear it on the live server (`set-hook -gu <hook>`, `set -pu -t <pane> <opt>`) or restart Рђћ otherwise the old behavior keeps firing. This repeatedly confused debugging of dynamic-styling work (each `set -p <style>` also triggers `recalculate_sizes` + a full redraw, so stale hooks cause flicker too).
- **nnn file-explorer popup (`M-d`) spans three layers.** `.tmux.conf` `M-d` Рєњ `tmux_scripts/tmux-nnn-explorer` (a `display-popup` running `nnn`) which threads `NNN_ORIGIN_PANE='#{pane_id}'` + `NNN_PLUG` into nnn's env; the `nnn/plugins/` scripts (`tmux-split-h`/`-v`, `cd-origin`, shared `.nnn-tmux-split`) read `$NNN_ORIGIN_PANE` to `split-window`/`send-keys` back into the *originating* window Рђћ a popup isn't in the window tree, so the pane id must be passed in. Changing the explorer means reading all three layers.
  - **`nnn/plugins/preview-tui`** is the vendored upstream plugin with **5 local patches** (re-fetching it clobbers all of them): (1) `shopt -s extglob 2>/dev/null` after shebang (bash 3.2 fix); (2) debounce `while read -r -t 0.15 newer`; (3) CURSEL dedup; (4) large-file guard (`--line-range=:500` for >500 KB); (5) **`fifo_pager()` tempfile rewrite** Рђћ renders preview content to `$PREVIEWFILE` first, then branches by pager: **moor** is fed the seekable regular file (`moor < $PREVIEWFILE`) so mouse-scroll reaches EOF without stalling; **less** (and any non-moor pager) is fed via a FIFO (`less < $FIFOPATH`), because a backgrounded `less` renders **blank** from a seekable file in the preview loop (navigation previews launch from the backgrounded `preview_fifo` subshell, where `less` grabs the tty during line-map init) Рђћ it only paints over a pipe/FIFO; moor buffers its input and paints fine either way.
  - **`nnn/plugins/fzcd`** is the vendored upstream plugin with **local patches** (re-fetching clobbers them): (1) `--preview` + `--preview-window 'right,60%,border-left'` added to both fzf calls (bat for files, eza tree for dirs); (2) zooms the nnn pane (`tmux resize-pane -Z`) while fzf runs so it fills the whole popup, restoring the split on exit via a trap (guarded by `#{window_zoomed_flag}`); (3) global Meta keys (`M-q` abort, `M-j/M-k` move, `M-u/M-n` preview scroll, `M-g` jump to `fzrg`); (4) `bat --theme=ansi` + large-file guard (>500KB Рєњ `--style=plain --line-range=:300`).
  - **`nnn/plugins/fzrg`** has **local patches**: global Meta keys (same set as fzcd, `M-g` reloads search), `bat --theme=ansi`, `RG_COLORS` match underline style. Re-fetching clobbers these.
  - **`tmux_scripts/tmux-nnn-explorer`** forwards `FZF_DEFAULT_OPTS` and `RG_COLORS` into the nested `nnn` tmux session via `-e` env vars. Without this, `;f`/`;g` fzf colors differ from other fzf surfaces. Also selects the pager at launch: uses `moor --no-linenumbers --mousemode=scroll --statusbar=plain` if `moor` is installed (`brew install moor`), falls back to `less -R --mouse --wheel-lines=3 -+F`. `--statusbar=plain` makes moor's statusbar **and** its `---` EOF marker use the terminal default background (transparent) instead of the default `inverse` reverse-video (light gray). Also passes `nnn -l 1` for 1-line-per-notch wheel scroll in the listing (`-l` = lines per mouse-wheel notch; lower = finer/smoother, default 1).
- **extrakto (`M-K`) launch path + vendored patch.** `M-K` Рєњ `tmux_scripts/tmux-extrakto-launch` (sets `TMUX_OPEN_PANE` so the shared opener `tmux-open-target` can resolve/split off the origin pane, mirroring `NNN_ORIGIN_PANE`) Рєњ vendored `tmux_plugins/extrakto/scripts/open.sh` Рєњ `extrakto_plugin.py` in a popup. `.tmux.conf` sets `@extrakto_fzf_unset_default_opts "false"` (keep the fzf palette) and `@extrakto_open_tool` (= `tmux-open-target`). **`extrakto_plugin.py` has 1 local patch (re-fetching clobbers it):** `get_option_only` was rewritten to batch **all** global options via one `tmux show-options -g` into a process-lifetime cache (`_OPTION_CACHE` / `_load_options`), instead of forking `tmux show-option -gqv <name>` once per option (~28 reads). The per-fork form cost ~330ms of pure startup latency before fzf appeared; batched is ~40ms. The cache unquotes/РђІunescapes values to match `-gqv` exactly, and returns `""` for unset options (unset options aren't listed by `show-options -g`) Рђћ identical contract, verified zero mismatches. The remaining variable cost is `@extrakto_grab_area "window full"` (default): captures up to `@extrakto_history_limit` (2000) lines from **every** pane in the window and tokenizes it all. If launch is still slow in a many-pane window with deep scrollback, narrow it (`recent` = visible viewport only / `full` = trigger pane full history) Рђћ that's a behavior change (which tokens are extractable), so it's a config choice, not applied by default.
- **Ghostty config overrides theme.** Keys in `ghostty/config` win over values in `themes/<name>`. Set palette overrides (`background`, `foreground`) in `config`, not in the theme file.
- **Ohmyposh foreground tokens.** The git template uses inline `<#hex>Рђд</>` for the icon/quotes while the segment's main `foreground:` covers the branch name. Both surfaces need updates when changing the git segment's palette.
- **tmux session order is fixed: `float` first, then creation order.** Sessions must be loaded and displayed in one consistent sequence across every surface Рђћ the `float` popup session leading, then the rest in the order they were opened. This mirrors the `t` bootstrap in `.zshrc` (`tmux new -d -s float` Рєњ `tmux new -A -s 'm*'`), so `m*` is normally second. `tsave` records this by capturing each session's `#{session_id}` (a monotonic `$N` creation counter) and `sort_by`-ing groups on it Рђћ **not** alphabetically, since jq `group_by` would otherwise sort by name. `tload` loads in that saved order and stable-sorts `float` to the front (a no-op when the snapshot is already creation-ordered) and attaches to the first non-`float` session. The status-right star list (`#{S:Рђд}` in `.tmux.conf`) and any other session-listing/switching/rendering must present the same float-first, creation-ordered sequence. Preserve this everywhere when touching session save/restore, listing, switching, or status rendering.

## tmux fzf navigation (`tmux-fzf-nav`)

- **`M-w`/`M-s`/`M-T`** are remade as fzf popup pickers via `tmux_scripts/tmux-fzf-nav`. Wrapper scripts (`tmux-window-ls`, `tmux-session-ls`, `tmux-other-panes-ls`) just `exec` into it Рђћ keep them thin so `.tmux.conf` bindings stay stable.
- **tmux `\t` in format strings is literal text, not a tab.** Use a shell-level separator (`SEP=$'\x1f'`) or print real tabs from `awk`/`printf`. Parsing tmux `-F '\tРђд'` output as tabs silently produces one field per row.
- **`display-popup -K` is not supported** by this tmux build (next-3.7) despite local man pages listing it. Using it makes picker launchers return 1 with `unknown flag -K`.
- **Popup/fzf `M-*` keys need two layers**: fzf `--bind alt-*` inside scripts, plus `.tmux.conf` conditional pass-through (`if -F '#{||:#{popup_width},#{==:#{session_name},nnn}}'`) so tmux root bindings don't intercept meta keys while a popup or nnn session is active.
- **`M-u`/`M-n` root binds must preserve normal copy-mode scroll outside popups.** Use conditional pass-through (`if-shell -F '#{||:#{popup_width},#{==:#{session_name},nnn}}'`), not unconditional `send-keys`. The else branch must use `run-shell` (not inline `copy-mode -e \;`) because `if-shell` brace form chokes on the semicolon.
- **Global fzf meta keys** (`M-q` abort, `M-j`/`M-k` move, `M-u`/`M-n` preview scroll, `M-g` enter/reload rg) are bound in every fzf surface: `tmux-fzf-nav` pickers + rg mode, `tmux-claude-menu`, `nnn/plugins/fzcd`, `nnn/plugins/fzrg`. When adding a new fzf surface, include the same set.
- **Pane-history ripgrep must use per-pane snapshot files, not a TSV.** nvim/opencode panes include ANSI, gutters, soft-wraps, duplicate visible line numbers Рђћ flattening into TSV rows breaks parsing and alignment. The rg flow writes `<pane_id>.txt` (plain, for rg) + `<pane_id>.ansi` (ANSI, for preview) + `meta.tsv` (paneРєњsession/window lookup) per search invocation.
- **For pane-history rg, "line" = tmux snapshot row**, not real filesystem/nvim buffer line. True file+line search would need a filesystem-aware mode.
- **rg menu match highlighting**: use `rg --color=always` with `--colors 'path:none' --colors 'line:none'` + explicit `--colors 'match:Рђд'` flags, and `fzf --ansi`. This keeps parsing reliable (path/line are uncolored) while highlighting only the match text.
- **rg preview must not use visible prefixes** (`>>` shifts nvim/opencode rows and breaks background colors). Use zero-width ANSI styling (`\e[4mРђд\e[24m`) around the matched snapshot row, re-applying after each embedded `\e[0m` reset.
- **`bat --theme=ansi` is the global bat theme** for all tmux/nnn fzf previews (`NNN_BATTHEME=ansi` in `.zshrc` + `tmux-nnn-explorer`). Keeps syntax colors aligned with the active terminal palette. The old `vague` TextMate theme is no longer used.


## "Building" / reloading

- **tmux**: `prefix r` (= `C-Space r`) Рђћ reloads `~/.config/.tmux.conf` (note the dot Рђћ the file is `.tmux.conf`, not `tmux/tmux.conf`).
- **Ghostty**: `cmd+shift+r` Рђћ reloads `~/.config/ghostty/config`.
- **zsh**: `source ~/.zshrc` (or open a new shell). Note: the file is `~/.config/.zshrc` symlinked or sourced; check the actual `$ZDOTDIR`.
- **nvim**: restart, or `:Lazy reload <plugin>` for hot-swappable changes.
- **ohmyposh**: re-render the prompt by pressing enter; no config-reload command needed.
- **AeroSpace**: `prefix r` (= `C-Space r`) Рђћ reloads `~/.config/aerospace/aerospace.toml`. (Service mode `r` also reloads; bind is `cmd-shift-enter` Рєњ `mode service` Рєњ `r`.)
- **SketchyBar**: `cmd-ctrl-alt-o` (Hammerspoon keybind Рђћ the sole reload path), or `brew services restart sketchybar` / `sketchybar --reload` from a shell. On monitor attach/detach SketchyBar destroys and recreates all bars and re-runs `sketchybarrc` Рђћ that's why startup cost matters there.
- **Hammerspoon**: reload its config (Hammerspoon menu Рєњ Reload Config, or `hs.reload()` via CLI). `hs -c "..."` invocations from AeroSpace use the live globals; a config reload picks up new `_G` exports.

## Files Claude should not edit unprompted

- `ghostty/themes/vague` and `ghostty/themes/oldworld` Рђћ base theme files; override in `ghostty/config` instead.
- `ohmyposh/base.json.bak` Рђћ backup of the prompt config.
- Any `*.bak` file.
- `nvim/lazy-lock.json` Рђћ managed by the Lazy plugin manager.
- Anything under `disabled-plugins/` in the nvim config Рђћ kept for reference, not loaded.
- `nvim/lua/utils/trouble.lua` Рђћ load-bearing helper; the folding and focus behavior of every `<leader>x*` keymap depends on it. If you must change it, do so deliberately and verify all mode keymaps still fold correctly.
- `sketchybar/sketchybarrc` (the bare file) Рђћ it's a plain-file copy that gets overwritten on each `cmd-ctrl-alt-p` toggle. Edit `sketchybarrc.full` / `sketchybarrc.performance` (the profile sources) instead.
